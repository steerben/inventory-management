System Architecture

Three-layer architecture. JSON files load once into Python memory at startup; all filtering happens in-process.

Data Flow — Inventory page, "London" warehouse filter

1
User selects warehouse in FilterBar selectedLocation ref from useFilters() composable updates → v-model on <select> in Inventory.vue triggers watcher.
2
Component calls API client loadData() reads getCurrentFilters() → { warehouse: "London", category: "all" } → passes to api.getInventory(filters).
3
Axios builds query string api.js constructs URLSearchParams, skips 'all' values, issues GET http://localhost:8001/api/inventory?warehouse=London.
4
FastAPI receives and filters main.py route get_inventory() calls apply_filters(inventory_items, warehouse, None) — list comprehension over in-memory data.
5
Pydantic validates response Decorator response_model=List[InventoryItem] serializes each dict against the model schema — type-safe JSON out.
6
Vue renders via computed Response assigned to inventoryItems.value → filteredItems computed property re-runs (search + sort) → v-for re-renders table.

Notable Patterns

Global filter state via composable singleton

useFilters() declares refs at module scope, so every component that imports it shares the same reactive state — no Pinia/Vuex needed.

Skip-'all' query param convention

API client omits parameters whose value is 'all', keeping URLs clean and letting the backend treat absence as "no filter."

Computed properties for derived data

Raw API data lives in ref(); search, sort, and aggregation all flow through computed() — automatic memoization until dependencies change.

Pydantic at every endpoint boundary

response_model=... on each route forces schema validation; broken JSON shape becomes a 500 instead of a silent client-side bug.

In-memory filtering in Python loops

apply_filters() uses list comprehensions over loaded dicts — appropriate for the demo's ~200–500 item dataset; would not scale to a real warehouse.

Multi-currency & i18n built-in

useI18n() + locale files (en.js, ja.js) and currency.js formatter support USD, JPY, EUR throughout the UI.
